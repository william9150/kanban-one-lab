# Kanban 看板管理系統

一個輕量級的 Kanban 看板網頁應用程式，讓使用者透過視覺化介面管理代辦事項。支援新增、編輯、刪除卡片，以及透過拖拉方式在不同狀態欄位間移動卡片。

## 功能特色

- **四欄位看板**：待處理、進行中、待驗收、已完成，每欄以狀態色彩標記區分
- **新增卡片**：輸入標題與描述，卡片自動加入「待處理」欄位
- **編輯卡片**：透過彈窗修改標題、描述與狀態，狀態變更時卡片自動移動至對應欄位
- **刪除卡片**：二次確認機制防止誤刪
- **拖拉移動**：跨欄拖拉卡片，狀態自動同步更新，支援拖放至欄位標題與空白區域
- **拖拉視覺回饋**：目標欄位顯示半透明高亮效果
- **即時通知**：卡片異動時右上角顯示提示訊息
- **資料自動儲存**：所有資料儲存在瀏覽器中，重新整理頁面後資料不會遺失
- **預設範例資料**：首次造訪自動載入 8 張範例卡片，方便快速體驗
- **響應式設計**：桌面 4 欄 / 平板 2 欄 / 手機 1 欄（可摺疊切換）
- **乾淨介面**：頁面本身不產生捲軸，僅欄位內部可滾動

## 技術選擇

| 類別     | 技術               | 選擇原因                                         |
| -------- | ------------------ | ------------------------------------------------ |
| 前端框架 | Vue 3 + TypeScript | TypeScript 提供型別檢查，減少開發時的錯誤        |
| 建置工具 | Vite               | 啟動速度快，修改程式碼後頁面即時更新，開發效率高 |
| 樣式方案 | Tailwind CSS       | 直接在 HTML 上寫樣式，快速完成響應式排版         |
| 狀態管理 | Pinia              | Vue 官方推薦的資料管理工具                       |
| 拖拉功能 | vue-draggable-plus | 拖拉套件，支援電腦滑鼠和手機觸控操作             |
| 資料儲存 | localStorage       | 不需要後端伺服器，資料直接存在瀏覽器中           |

## 安裝

確保已安裝 Node.js 18+，然後執行：

```bash
npm install
```

## 啟動開發伺服器

```bash
npm run dev
```

開啟瀏覽器前往 http://localhost:3000

## 建置生產版本

```bash
npm run build
```

打包產出位於 `dist/` 資料夾。

## 預覽生產版本

```bash
npm run preview
```
