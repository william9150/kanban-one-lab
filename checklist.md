# Kanban Board — 人工測試 Checklist

> 測試前準備：清除瀏覽器 localStorage（DevTools → Application → Local Storage → 刪除 `kanban-cards`），確保從初始狀態開始測試。

---

## 一、初始狀態與預設資料

- [ ] **TC-0.1** 首次造訪（localStorage 無資料）時，看板自動載入 8 張預設卡片
- [ ] **TC-0.2** 「待處理」欄位顯示 2 張卡片：「更新履歷表」、「準備作品集」
- [ ] **TC-0.3** 「進行中」欄位顯示 2 張卡片：「投遞履歷-ABC科技公司」、「準備面試問題」
- [ ] **TC-0.4** 「待驗收」欄位顯示 2 張卡片：「第一輪面試-XYZ企業」、「薪資談判-DEF新創」
- [ ] **TC-0.5** 「已完成」欄位顯示 2 張卡片：「完成面試準備」、「建立LinkedIn檔案」
- [ ] **TC-0.6** 每張卡片都顯示標題與描述內容

---

## 二、頁面結構與欄位顯示（User Story 2）

- [ ] **TC-1.1** 頁面頂部有「新增代辦事項卡片」專屬輸入區塊（標題欄位 + 描述欄位 + 新增按鈕）
- [ ] **TC-1.2** 卡片管理區塊包含四個明確劃分的獨立欄位：「待處理」「進行中」「待驗收」「已完成」
- [ ] **TC-1.3** 每個欄位標頭顯示：圓形色彩標記 + 欄位名稱 + 卡片數量
- [ ] **TC-1.4** 色彩標記正確：待處理（紅色）、進行中（橘色）、待驗收（藍色）、已完成（綠色）
- [ ] **TC-1.5** 將某欄位的所有卡片移走後，顯示「暫無卡片」空狀態提示（虛線邊框）
- [ ] **TC-1.6** 每張卡片只歸屬於四個狀態之一（不會同時出現在多個欄位）

---

## 三、新增卡片（User Story 1）

- [ ] **TC-2.1** 填寫標題與描述，點擊「新增卡片」→ 卡片立即出現在「待處理」欄位
- [ ] **TC-2.2** 新增成功後，標題與描述輸入欄位自動清空
- [ ] **TC-2.3** 僅填寫標題（描述留空），點擊新增 → 卡片成功建立，描述區域不顯示
- [ ] **TC-2.4** 標題欄位為空時，「新增卡片」按鈕呈 disabled 狀態，無法點擊
- [ ] **TC-2.5** 標題僅輸入空白字元時，「新增卡片」按鈕呈 disabled 狀態
- [ ] **TC-2.6** 新增卡片後，「待處理」欄位的卡片數量計數正確更新
- [ ] **TC-2.7** 連續新增多張卡片，每張都正確顯示在「待處理」欄位

---

## 四、編輯卡片（User Story 3）

### 4.1 Hover 行為
- [ ] **TC-3.1** 桌面版：滑鼠 hover 卡片時，右上角顯示「編輯」與「刪除」圖示按鈕
- [ ] **TC-3.2** 桌面版：滑鼠離開卡片時，按鈕隱藏
- [ ] **TC-3.3** 手機版（< 640px）：編輯/刪除按鈕常駐顯示，不需 hover

### 4.2 編輯 Modal
- [ ] **TC-3.4** 點擊「編輯」按鈕 → 開啟編輯 Modal 彈窗
- [ ] **TC-3.5** Modal 開啟時自動帶入該卡片的現有標題、描述、狀態
- [ ] **TC-3.6** Modal 包含：標題輸入欄、描述輸入欄、狀態下拉選單、儲存按鈕、取消按鈕
- [ ] **TC-3.7** 狀態下拉選單包含四個選項：待處理、進行中、待驗收、已完成
- [ ] **TC-3.8** 修改標題和描述 → 點擊「儲存」→ Modal 關閉，卡片內容立即更新
- [ ] **TC-3.9** 在 Modal 中將狀態從「待處理」改為「進行中」→ 儲存 → 卡片移至「進行中」欄位
- [ ] **TC-3.10** 在 Modal 中將狀態改為其他三個狀態 → 各自正確移動到對應欄位
- [ ] **TC-3.11** 點擊「取消」按鈕 → Modal 關閉，卡片內容不變
- [ ] **TC-3.12** 點擊 Modal 外部遮罩區域 → Modal 關閉，卡片內容不變
- [ ] **TC-3.13** 按下 ESC 鍵 → Modal 關閉，卡片內容不變
- [ ] **TC-3.14** Modal 顯示時，背景有半透明黑色遮罩

---

## 五、刪除卡片

- [ ] **TC-4.1** 點擊「刪除」按鈕 → 跳出確認對話框
- [ ] **TC-4.2** 確認對話框顯示卡片名稱與警告訊息
- [ ] **TC-4.3** 點擊「取消」→ 對話框關閉，卡片仍然存在
- [ ] **TC-4.4** 點擊「確認刪除」→ 卡片從看板中移除
- [ ] **TC-4.5** 刪除後，該欄位的卡片數量計數正確更新
- [ ] **TC-4.6** 刪除欄位中最後一張卡片後，顯示空狀態提示

---

## 六、拖拉移動卡片（User Story 4）

### 6.1 跨欄拖拉
- [ ] **TC-5.1** 滑鼠按住卡片並拖動 → 卡片跟隨滑鼠移動
- [ ] **TC-5.2** 拖動過程中，原位置顯示佔位效果（半透明）
- [ ] **TC-5.3** 將卡片從「待處理」拖到「進行中」→ 卡片停留在「進行中」
- [ ] **TC-5.4** 將卡片從「進行中」拖到「待驗收」→ 卡片停留在「待驗收」
- [ ] **TC-5.5** 將卡片從「待驗收」拖到「已完成」→ 卡片停留在「已完成」
- [ ] **TC-5.6** 將卡片從「已完成」拖回「待處理」→ 卡片停留在「待處理」
- [ ] **TC-5.7** 將卡片拖到空的欄位 → 卡片正確進入該空欄位
- [ ] **TC-5.8** 跨欄拖拉後，兩個欄位的卡片數量計數正確更新

### 6.2 同欄排序
- [ ] **TC-5.9** 在同一欄位內拖動卡片 → 卡片排序更新
- [ ] **TC-5.10** 同欄排序後，卡片數量不變

### 6.3 拖拉動畫
- [ ] **TC-5.11** 拖拉過程中有平滑的過渡動畫

---

## 七、資料持久化

- [ ] **TC-6.1** 新增卡片後，重新整理頁面 → 新增的卡片仍然存在
- [ ] **TC-6.2** 編輯卡片後，重新整理頁面 → 編輯內容已保存
- [ ] **TC-6.3** 刪除卡片後，重新整理頁面 → 被刪除的卡片不再出現
- [ ] **TC-6.4** 拖拉卡片到新欄位後，重新整理頁面 → 卡片仍在新欄位
- [ ] **TC-6.5** 打開 DevTools → Application → Local Storage → 確認 `kanban-cards` key 存在且資料正確
- [ ] **TC-6.6** 手動損壞 localStorage 資料（將值改為無效 JSON）→ 重新整理頁面 → 應用正常運作，不會白畫面
